### Why Dapr?

1. **Language Agnostic** - Works with any language (Python, Go, Java, Node.js, etc.)
2. **Platform Agnostic** - Runs on Kubernetes, containers, local, edge
3. **Cloud Agnostic** - Works with Azure, AWS, GCP, and on-premises
4. **Sidecar Pattern** - No code changes needed to use Dapr capabilities
5. **Built-in Best Practices** - Retry, timeout, circuit breaker, tracing, metrics

---

## Prerequisites

- **Python**: 3.9 or higher
- **uv**: Fast Python package manager
- **Docker**: For running Dapr components
- **Dapr CLI**: For managing Dapr runtime

---

## Quick Start with uv

### 1. Install uv

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 2. Verify Installation

```bash
uv --version
# Expected output: uv 0.x.x
```

### 3. Create Project Structure

```bash
# Create the tutorial project
mkdir -p dapr-python-tutorial
cd dapr-python-tutorial

# Initialize uv project
uv init --name dapr-tutorial --app

# Create module structure
mkdir -p modules/{01-intro,02-service-invocation,03-state-management,04-pubsub,05-bindings,06-actors,07-secrets,08-observability,09-advanced}
mkdir -p examples/{hello-world,ordering-service,chat-app}
mkdir -p docs/diagrams
```

### 4. Install Dapr SDK

```bash
# Add Dapr SDK as dependency
uv add dapr

# Add useful packages
uv add rich typer pydantic
uv add --dev pytest httpx
```

## Quick Start Example

### 1. Start Dapr Sidecar

```bash
dapr run --app-id myapp --dapr-http-port 3500 -- python main.py
```

### 2. Minimal Dapr Python App

```python
from dapr.clients import DaprClient

with DaprClient() as dapr:
    # Save state
    dapr.save_state(
        store_name="statestore",
        key="mykey",
        value="Hello Dapr!"
    )

    # Get state
    data = dapr.get_state(store_name="statestore", key="mykey")
    print(data.json())  # Output: "Hello Dapr!"
```
